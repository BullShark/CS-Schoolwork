COSC2415 Data Structures Spring 2006

Programming Assingment 1: The Bag ADT

Due on 2/15/2006

This programming assignment relys on knowledge and skills you develop from In-class exercises and chapters 1-3.

I. Program Descritpion:

A bag is very similar construct to a set except that duplicates are allowed in a bag. Implement a bag collection by creating both a BagADT<T> interface and a ArrayBag<T> class. The BagADT<T> extends the SetADT<T>, so ArrayBag<T> inherits all the methods specified in SetADT<T> as well as three additional methods specified in BagADT<T> . The three additional operations to be included in BagADT are:
  1. The difference method: This operation would take a bag as a parameter and subtract the contents of that bag from the current bag if they exist in the current bag. The result would be returned in a new bag. Be careful to consider possible exceptional situations.
  2. The intersection method: This operation would take a bag as a parameter and would return a bag containing those elements exist in both bags. Be careful to consider possible exceptional situations.
  3. The count method: This operation would take an element as a parameter and return the number of copies of that element in the bag. Be careful to consider possible exceptional situations.

II. The Implementation:

You will design and implement the BagADT<T> interface and the ArrayBag<T> class stated above.

Write a MusicCD class that contains a CD title and sale price. You can find out the CD title or the sale price of a music CD and you can find out whether or not another MusicCD object is the same.

Write a MusicCDBagTester class that creates two ArrayBag objects. Your program will read in music CD information from files musicCD1.txt and musicCD2.txt and store the information in to the two ArrayBag objects. The MusicCD class is passed as generic data type for ArrayBag object. Your program should test and output all the functionality of your the ArrayBag class.

Give Big O analysis based on your implementation on every operation listed in the BagADT.

III. Hand-In:

  1. A UML diagram of your design. It should be drawn using PowerPoint or equivalent. (10 points)
  2. Your source codes listings. (70 points)
  3. Output of your program testing. (10 points)
  4. Big O Analysis for your implementation of each method in the BagADT interface. (10 points)
  5. One page, single space report describe your project. What's working what's not working, and what should I look for.
